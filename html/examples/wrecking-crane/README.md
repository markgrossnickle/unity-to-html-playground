# Wrecking Crane

A Phaser + **Matter.js** physics playground. A side-view construction crane
on the left, a brick house on the right. Drag the crane's arm to swing the
wrecking ball, hit the house, watch the pieces tumble.

## What this exercises

- **Phaser.Physics.Matter** — a dynamic Matter circle (the ball) is tethered
  to an invisible static anchor by a single Matter constraint
  (`length=220, stiffness=0.85, damping=0.04`). The anchor is teleported to
  the arm-tip world position every frame, so the ball pendulum-swings
  beneath whatever angle the user has set the arm to.
- **Direct-control arm + impulse-on-release.** The arm sprite is not
  physics-simulated. The scene tracks `armAngle` directly, clamps it to
  a -126° to +29° range from horizontal, and computes the arm's angular
  velocity per-frame so that on `pointerup` it can inject a tangential
  impulse into the ball — flicking the arm gives a satisfying swing.
- **Demolition.** House pieces are built as **static** Matter rectangles.
  On `collisionstart` with the ball, if the ball's speed exceeds
  `SHATTER_VELOCITY`, the piece flips to dynamic with `Body.setStatic(false)`
  and gets an impulse force away from the ball. Upper pieces lose their
  support and cascade naturally.
- **Chain visualization.** A single drawn line from arm-tip to ball with
  small chain-link beads. We did NOT model the chain as a chain of linked
  Matter circles — that adds 12+ bodies and constraints to tune. See
  `src/chain.ts`.

## Files

```
examples/wrecking-crane/
├── index.html       page chrome + reset button
├── main.ts          Phaser bootstrap (Matter physics enabled)
├── style.css        toolbar + game CSS
├── src/
│   ├── CraneScene.ts  the scene — geometry, input, collision, reset
│   ├── house.ts       house piece layout (~8 rectangles)
│   └── chain.ts       chain rendering
└── assets/          PNGs generated by scripts/generate-crane-assets.mjs
```

## Regenerating art

```sh
npm run gen-crane-assets
```

This rebuilds the sky background, the crane (lattice tower + horizontal arm
with hook fitting), the wrecking ball, and all eight house-piece sprites.
All art is drawn programmatically with `node-canvas`.
