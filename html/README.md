# html — Phaser playground

Vite + TypeScript + Phaser 3. Hosts the runtime that plays back animations exported from Unity (see `../unity/`), plus a landing page that lists self-contained examples.

## Run

```sh
cd html
npm install
npm run dev
```

Open the printed URL. The landing page lists each example.

## Build for production

```sh
npm run build       # tsc --noEmit + vite build
npm run preview     # serve dist/ locally
```

The output is plain static files in `dist/`. Drop them on any static host.

## Layout

```

├── index.html                        landing page (lists examples)
├── src/
│   ├── landing/                      landing-page CSS + entry module
│   └── runtime/
│       └── PhaserAnimPlayer.ts       shared runtime — every example imports this
├── examples/
│   ├── animation-playback/           one folder per example
│   │   ├── index.html                Vite entry
│   │   ├── main.ts                   bootstraps a Phaser.Game
│   │   ├── style.css                 example-local CSS
│   │   └── assets/                   generated by scripts/gen-sample-assets.mjs
│   ├── coloring-book/                tap-to-fill line-art pictures
│   ├── dino-drop/                    Matter physics + brontosaurus tail-slide
│   ├── number-tracer/                drag-to-trace digits 0–9 on dotted paths
│   ├── sound-pads/                   4×4 launchpad — Web Audio synth, multi-touch
│   ├── space-dodge/                  vertical-scrolling asteroid dodger
│   ├── sticker-board/                drag/rotate/scale stickers (localStorage)
│   └── wrecking-crane/               Matter physics + draggable crane arm
├── scripts/
│   ├── gen-sample-assets.mjs         regenerates the playback example's assets
│   ├── generate-coloring-assets.mjs  regenerates coloring-book art
│   ├── generate-dino-assets.mjs      regenerates the brontosaurus + drawer toys
│   ├── generate-sticker-assets.mjs   regenerates the 12 cartoon stickers
│   ├── generate-crane-assets.mjs     regenerates the crane + house pieces
│   └── generate-space-assets.mjs     regenerates the ship + asteroids + heart
└── vite.config.ts                    multi-page entry table
```

## Adding a new example

1. Create `examples/<name>/index.html` and `examples/<name>/main.ts`.
2. Register the entry in `vite.config.ts` under `build.rollupOptions.input`.
3. Add a card to `index.html` linking to `/examples/<name>/index.html`.

That's it — Vite serves it automatically in dev, builds it as a separate page in production.

## Regenerating the sample assets

Each example with programmatically-drawn art has its own generator. They all
write into the example's `assets/` folder.

```sh
npm run gen-sample-assets    # animation-playback
npm run gen-coloring-assets  # coloring-book
npm run gen-dino-assets      # dino-drop
npm run gen-sticker-assets   # sticker-board
npm run gen-crane-assets     # wrecking-crane
npm run gen-space-assets     # space-dodge
```

`gen-sample-assets` writes `examples/animation-playback/assets/{atlas.png,
atlas.json, animation.json}`.

## Replacing the sample with a real Unity export

See `../unity/EXPORT_GUIDE.md`. End state:

```
examples/animation-playback/assets/
  animation.json   ← from PhaserAnimExporter
  atlas.png        ← from TexturePacker
  atlas.json       ← from TexturePacker (Phaser 3 JSON-Hash)
```

The runtime is format-stable, so re-exporting in place is a hot-reload-friendly loop.
